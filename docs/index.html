Installation

Unzip the download package. All the files of the website are in the HTML folder. You can simply copy the HTML folder itself to your web server or localhost. Once installed, visit "/index.html" to view the website.

HTML Structure

Smooth Sailing is built with the Bootstrap 3 framework - http://www.getbootstrap.com - that means you have access to all the great features, css & js goodies has to offer. To learn more about Bootstrap's grid system, please refer to http://getbootstrap.com/css/#grid

There is one single html document, where all the HTML for the site is present - this file is called index.html. The different sections of the website are clearly separated in commented sections, in the HTML and CSS files, to help you find your way easily through the files.

CSS Structure

Smooth Sailing was developed using LESS. The .less files are included in the template, although not used directly for the website. They allow you to customise the template via less if you wish to. To do so, you'll need to compile those files into the respective css files.

There are 6 color-skins and therefore 6 stylesheets, named style1, style2, etc. You'll need to select which one of these files you want to work with and link it in the index.html head section. By default, the style1 stylesheet is linked.

Using LESS files has obvious advantages - for example you can change the primary highlight color (#0099aa) by going into the /less/variables.less file and changing the value of the "@highlight-one" variable. Changing the color there and saving the file, providing you're compiling correctly, will instantly change all the instances where this color is used, and you will have a brand new color scheme!

The index.html file calls one single css file, style.css. Style.css imports a few other css files, including the bootstrap css file. If you customise the css, you should place your css in style.less / style.css and leave the other files as is.

Javascript

In the /js/ folder, you'll find the custom.js which is the file where scripts are written. You can modify this file, but the other js files should, in most cases, be left alone.

With a bit of jQuery knowledge, in custom.js you'll be able to tweak the css animations on scroll, the twitter widget and the newsletter signup ajax form (see below).

Newsletter Signup Form

To have the newsletter form working, you'll need to host your website on an APACHE server that can handle PHP files correctly. The main thing you need to do to get the form to work is to enter your desired e-mail address in the config.php file. This is where the notification of a new signup will be e-mailed. Please note that you'll need to create manually your mailing list with the incoming subscription e-mails.

The config.php file is in the root directory of the website and contains one line. Replace "your-email-address@here.com" with the appropriate address, that's all you need to do!

If you want to customise the error / success / notification messages, you can do so in /js/custom.js (for the success message) and in /assets/newsletter.php (for the error messages & e-mail confirmation).

Remember: this form will not work if server doesn't handle PHP files.

Contact Form

Works exactly like the newsletter signup form described above, except that the script responsible for this form is in /assets/contact.php

If you want to customise the error / success / notification messages, you can do so in /js/custom.js (for the success message) and in /assets/contact.php (for the error messages & e-mail confirmation).

Remember: this form will not work if server doesn't handle PHP files.

Twitter Widget

To get your twitter feed working, you'll need to put your own twitter username in the settings. You can find the "Twitter Feed" section in /js/custom.js almost at the bottom of the file. Find the line "username" that has "envato" as a demo, and place your username id here instead. You can play around with other settings, such as display more than 1 tweet, change the "a few hours ago" text, etc.

For full documentation on this twitter widget, please refer to https://github.com/StanScates/Tweet.js-Mod.
